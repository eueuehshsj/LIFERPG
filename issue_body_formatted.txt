## 개요

`Task.md`의 "추가 수정 사항 2" 항목을 정식 이슈로 정리합니다.

1) 퀘스트·보상 로컬 스토리지 저장
2) 하단 버튼을 화면 비율에 따라 유연하게 변동하도록 Flex로 수정

## 현재 동작

• 로컬 스토리지에 데이터를 저장하지 않아 새로고침 시 모든 데이터가 초기화됩니다. (참고 이슈: #61)
• 하단 가판대 버튼들이 절대 위치/고정 크기 기반으로 구현되어 화면 비율이나 해상도 변경 시 정렬 및 크기 문제가 발생합니다.

## 목표 동작

• 퀘스트·보상 관련 상태(`tasks`, `completedTasks`, `rewards`, `spentPoints`)를 `localStorage`에 저장하고 앱 초기화 시 복원합니다.
• 하단 버튼 그룹을 Flex 기반 반응형 레이아웃으로 전환하여 화면 크기·비율 변화에 따라 버튼의 배치와 크기가 자연스럽게 조정됩니다. 모바일/태블릿/데스크탑에서 사용성 유지.

## 수정 대상

• `src/app/App.tsx` — 상태 초기화 및 `useEffect` 저장 로직 추가
• `src/app/components/*` — `AddTaskModal.tsx`, `AddRewardModal.tsx`, `RewardClaimModal.tsx` 등 포인트/보상 관련 컴포넌트
• 하단 가판대 관련 컴포넌트/스타일 — 버튼 컨테이너를 Flex로 변경

## 구현 방식

• `localStorage` 키: `liferpg_tasks`, `liferpg_completedTasks`, `liferpg_rewards`, `liferpg_spentPoints`.
  - `useState`의 lazy initializer로 `localStorage`에서 초기값 읽기
  - 상태 변경 시 `useEffect`로 `JSON.stringify` 하여 저장
  - 파싱 에러 대비 try/catch, 데이터 포맷 유효성 검사

• 하단 버튼 반응형 전환
  - 버튼 컨테이너를 CSS Flexbox로 전환 (`display: flex`, `flex-wrap`, `gap`, `justify-content` 등)
  - 버튼은 `flex-basis`/`min-width`/`max-width`로 크기 범위 지정
  - 작은 화면에서는 2~3줄로 감싸지도록 `flex-wrap` 사용, 또는 미디어쿼리로 세로 스택 전환
